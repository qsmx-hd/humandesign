# 🚀 Human Design AI — GitHub Pages 部署指南

## 第一步：创建 GitHub 仓库

1. 打开浏览器，访问 [github.com](https://github.com)，登录你的账号
2. 点击右上角 **"+"** 号 → 选择 **"New repository"**
3. 填写仓库信息：
   - **Repository name**: `human-design-ai`（小写，用短横线连接）
   - **Description**: 人类图智能助手 - Human Design AI
   - **Public** ✅（选公开，这样才能用免费的 GitHub Pages）
   - **Add a README file** ✅（勾选上）
4. 点击 **"Create repository"**

---

## 第二步：上传文件到仓库

### 方法 A：使用网页直接上传（最简单，推荐新手）

1. 在你刚创建的仓库页面，点击 **"Add file"** → **"Upload files"**
2. 把以下 **5 个文件** 从电脑拖拽到上传区域：
   - `index.html`
   - `chat.html`
   - `beginner.html`
   - `intermediate.html`
   - `advanced.html`
3. 在页面底部的 **"Commit message"** 中输入：`初始化：添加 Human Design AI 网站文件`
4. 点击绿色按钮 **"Commit changes"**

> 📂 文件来源路径：`c:\Users\user\WorkBuddy\20260404003959\`

### 方法 B：使用 Git 命令行上传

如果你之前学过 Git 命令行，可以用这个方法：

```powershell
# 1. 进入你的工作区
cd C:\Users\user\WorkBuddy\20260404003959

# 2. 初始化 Git 仓库
git init

# 3. 添加远程仓库（把 YOUR_USERNAME 替换成你的 GitHub 用户名）
git remote add origin https://github.com/YOUR_USERNAME/human-design-ai.git

# 4. 添加所有文件
git add index.html chat.html beginner.html intermediate.html advanced.html

# 5. 提交
git commit -m "初始化：添加 Human Design AI 网站文件"

# 6. 推送到 GitHub
git branch -M main
git push -u origin main
```

---

## 第三步：开启 GitHub Pages

1. 进入你的仓库页面
2. 点击顶部标签栏的 **"Settings"**（设置）
3. 在左侧菜单中找到 **"Pages"**（在 "Code and automation" 分类下）
4. 在 "Build and deployment" 部分：
   - **Source**: 选择 **"Deploy from a branch"**
   - **Branch**: 选择 **"main"** 分支，文件夹选 **"/ (root)"**
5. 点击 **"Save"**

---

## 第四步：等待部署完成

- 页面会显示：**"Your site is ready to be published at..."**
- 通常需要 **1-3 分钟** 部署完成
- 刷新页面，看到绿色提示框 **"Your site is published at..."** 就成功了！

你的网站地址将是：

## 🔧 常见问题

### Q: 部署后页面显示 404？
- 确认文件名是 **小写** 的 `.html`
- 等待 2-3 分钟让部署完成
- 检查 URL 拼写是否正确

### Q: chat.html 的 AI 不回复？
- 打开浏览器 **F12 开发者工具** → **Console** 查看错误
- 最可能的原因：DeepSeek API Key 需要充值或有余额
- 检查：登录 [DeepSeek 开放平台](https://platform.deepseek.com/) 查看 API 余额

### Q: 如何更新网站？
- 只需修改本地文件，然后重新上传（或用 git push）
- GitHub Pages 会自动重新部署

## 📌 重要提醒

- **API Key 安全**：当前 API Key 是写在前端代码中的，任何人查看网页源码都能看到。建议在 DeepSeek 平台设置 **API 调用限额**，防止被滥用。
- **费用控制**：DeepSeek API 按 token 计费，价格很低，但建议设置月度预算上限。
